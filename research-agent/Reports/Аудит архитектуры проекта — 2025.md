### 2.2. .claude/settings.local.json — настройки Claude Code

```json
{
  "env": {
    "DEFAULT_SEARCH_LANG": "ru",
    "RESEARCH_DIR": "/Users/liza/research-agent"
  },
  "permissions": {
    "allow": [
      "WebSearch(*)", "WebFetch(*)",
      "Bash(find:*)", "Bash(ls:*)", "Bash(cat:*)", "Bash(grep:*)"
    ],
    "deny": [
      "Bash(git push:*)", "Bash(git push --force:*)",
      "Bash(curl:--request DELETE*)"
    ]
  },
  "enabledMcpjsonServers": ["brave-search"],
  "enableAllProjectMcpServers": true
}
```

**Ключевые настройки:**
- Язык поиска по умолчанию — русский
- Разрешены WebSearch, WebFetch и read-only Bash-команды
- Запрещены git push и DELETE-запросы через curl
- Включён MCP-сервер brave-search

## 4. Оставшиеся проблемы

### 4.1. Пустой API-ключ в .mcp.json

Поле `BRAVE_API_KEY` пустое. MCP-сервер `brave-search` не сможет работать без ключа. При этом в `settings.local.json` он отмечен как включённый.

**Серьёзность:** средняя (сервер не работает, но и не мешает)

**Рекомендация:** Получить API-ключ Brave Search и внести его в `.mcp.json`, либо удалить конфигурацию сервера и убрать `brave-search` из `enabledMcpjsonServers`.

### 4.2. .DS_Store в корне проекта

Системный файл macOS `.DS_Store` присутствует, но теперь исключён через `.gitignore`. При подключении Git он не попадёт в историю.

**Серьёзность:** низкая (смягчено `.gitignore`)

**Рекомендация:** Удалить файл вручную при желании очистить структуру, иначе он игнорируется.

### 4.3. Несогласованность регистра имён папок

Папки называются с заглавной буквы: `Archive`, `Reports`, `Results`. В правиле нейминга файлов используется строчный стиль с дефисами, но для папок стиль не задан. Это может привести к путанице при обращении к путям в разных ОС.

**Серьёзность:** низкая

**Рекомендация:** Унифицировать — либо все папки со строчной буквы (`archive`, `reports`, `results`), либо добавить описание текущей конвенции в `claude.md`.

### 4.4. Пустая папка Archive

`Archive/` — пустая папка. Назначение описано в `claude.md`, но не формализовано: какие файлы и когда перемещать в архив?

**Серьёзность:** низкая

**Рекомендация:** Добавить правило архивации в `claude.md` (например: «Перемещать файлы в Archive/ по запросу Лизы или через 30 дней после создания»), либо добавить `README.md` в `Archive/`.
